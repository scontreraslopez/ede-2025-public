# Temario EDE - CFGS DAM / DAW

## IES Severo Ochoa - Curso 2025-26

Este repositorio contiene los materiales didácticos de la asignatura **Entornos de Desarrollo (EDE)** para el Ciclo Formativo de Grado Superior en Desarrollo de Aplicaciones Multiplataforma (DAM) y Desarrollo de Aplicaciones Web (DAW).

---

## Contenido del Repositorio

El temario está organizado en unidades de aprendizaje (UP - Unidad de Programación). Cada unidad incluye:

- **Contenidos teóricos**: Explicaciones conceptuales y ejemplos prácticos
- **Recursos externos**: Enlaces a documentación y herramientas complementarias
- **Actividades**: Prácticas para consolidar los conocimientos
- **Material de ampliación**: Contenido opcional para profundizar (no evaluable)

---

## Unidades Disponibles

### UP06: Fases en el Desarrollo del Software

**Contenido evaluable:**

- Ciclo de vida del software (SDLC): fases, modelos y metodologías
- Modelo en cascada y sus variantes
- Metodologías ágiles: Scrum, Kanban y gestión iterativa
- Mantenimiento y evolución del software

[Ir a UP06](./UP06/UP06.md)

---

### UP07: Principios de Diseño Orientado a Objetos

**Contenido evaluable:**

- Principios de la Programación Orientada a Objetos (POO)
- Principios SOLID: responsabilidad única, abierto/cerrado, sustitución de Liskov, segregación de interfaces, inversión de dependencias
- Patrones de diseño básicos y su relación con SOLID
- Refactorización orientada a principios de diseño

[Ir a UP07](./UP07/UP07.md)

---

### UP08: UML Estructural

**Contenido evaluable:**

- Diagramas de casos de uso: actores, casos, relaciones `include` y `extend`
- Diagramas de clases: clases, atributos, métodos, relaciones (herencia, composición, agregación, dependencia)
- Diagramas de objetos: instancias y estado en un momento concreto
- Herramientas para modelado UML (PlantUML, draw.io)

[Ir a UP08](./UP08/UP08.md)

---

### UP09: UML de Comportamiento

**Contenido evaluable:**

- Diagramas de secuencia: participantes, mensajes, activaciones
- Diagramas de comunicación: objetos y mensajes etiquetados
- Diagramas de actividades: flujos, decisiones, concurrencia
- Diagramas de estados: estados, transiciones, condiciones de guarda
- Diagramas como código: PlantUML y Mermaid

[Ir a UP09](./UP09/UP09.md)

---

### UP10-11: Diseño y Ejecución de Pruebas

**Contenido evaluable:**

- Tipos de pruebas: unitarias, de integración, funcionales
- JUnit 5: estructura, ciclo de vida, aserciones
- Tests parametrizados y tests de excepción
- Dobles de prueba: stubs, mocks y Mockito
- Desarrollo dirigido por pruebas (TDD): ciclo Red-Green-Refactor

[Ir a UP10-11](./UP10-11/UP10-11.md)

---

### UP12: Optimización y Refactorización de Código

**Contenido evaluable:**

- Concepto de refactorización: cuándo y cómo aplicarla
- Code smells: síntomas de código con problemas de diseño
- Técnicas de refactorización en IntelliJ IDEA
- Análisis estático con SonarQube for IDE
- Integración continua (CI) como mecanismo de calidad

[Ir a UP12](./UP12/UP12.md)

---

### UP13: Documentación de Código

**Contenido evaluable:**

- Importancia y objetivos de documentar código
- Javadoc: etiquetas, estructura, generación con IntelliJ
- Comentarios eficaces: qué documentar y qué no
- Generación automática de documentación

[Ir a UP13](./UP13/UP13.md)

---

## Cómo Usar Este Repositorio

1. **Lectura del contenido**: Cada archivo `.md` contiene la teoría y ejemplos prácticos
2. **Práctica**: Se recomienda reproducir los ejemplos con las herramientas indicadas en cada unidad
3. **Actividades**: Las actividades evaluables se publican en Aules
4. **Ampliación**: El contenido marcado como ampliación es opcional y no entra en la evaluación ordinaria

---

## Herramientas Recomendadas

- **IDE principal**: IntelliJ IDEA (Community o Ultimate)
- **Control de versiones**: Git + GitHub
- **Modelado UML**: PlantUML, draw.io
- **Diagramas como código**: [PlantUML online](https://www.plantuml.com/plantuml/), [Mermaid live](https://mermaid.live/)
- **Análisis de calidad**: SonarQube for IDE (plugin para IntelliJ)
- **Testing**: JUnit 5 + Mockito (integrados en IntelliJ)
